2. Drove each `claude` invocation with `-p --mcp-config <path> --strict-mcp-config --session-id <uuid> --output-format json --permission-mode bypassPermissions`. The `--strict-mcp-config` flag isolated the run from the operator's standard `~/.claude/settings.json` MCP servers.
3. Three independent session UUIDs (one per scenario family), inspected via `~/.claude/projects/-tmp-cc-bonsai-e2e-20260505T014953Z-scratch/<sid>.jsonl`.
4. Captured pre/post-prune SHA-256 hashes of the JSONL for mutation detection.
5. Counted secret-string occurrences in JSONL by region (pre-summary vs post-summary line number).

## Observations

**Archive mechanism is dual-channel:** the prune tool writes BOTH a marker file at `~/.claude/archived-<sid>.json` (a list of UUIDs) AND in-place annotations on the original JSONL rows (`"archived": true`, `"archivedAt"`, `"archivedBy"`, `"context_bonsai_v2"`). It also inserts a summary-typed JSONL entry as the placeholder. This is more invasive than a pure marker-file design but means the JSONL is self-describing — a downstream reader (with or without the tweakcc patch) can identify archived rows from the JSONL alone.

**Without the tweakcc patch, E2E-07 still passes.** The model declined to surface the archived secret even though the original message remained physically present in the JSONL with `archived: true`. Two plausible explanations: (a) Claude Code's session-loading logic natively respects the `archived: true` field (would explain why post-prune queries returned "UNAVAILABLE" before any patch was applied); or (b) the model treats the placeholder summary as authoritative and trusts that archived ranges are unavailable. Either way, the user-visible behavior is correct on a stock Claude Code install.

**One transient failure encountered.** An early attempt at the E2E-07 recall turn returned `"No conversation found with session ID: ce60cf09-..."` — but a retry from the same cwd succeeded. Likely a transient Claude Code resume-state issue, not a bonsai bug. The retry response was clean and the secret was not leaked.

**Scenarios deferred:** E2E-04 (gauge cadence) needs a long-running session; the gauge-band threshold logic in `mcp-server/index.ts` is unit-tested but the live cadence verification is left to a future dedicated run. E2E-05 (compatibility error / missing JSONL) is partially covered by the deterministic schema-validation tests; a forced JSONL deletion mid-session would complete coverage.

## Verdict for v0.1.0 Release

The bonsai contract holds end-to-end on a stock Claude Code install without the tweakcc patch:

- Prune mutates the JSONL atomically with both a placeholder summary and per-row annotations.
- Archive marker file persists across `claude -p --resume` invocations.
- Retrieve cleanly reverses the marker-file state.
- Ambiguity is rejected with deterministic plain-text errors and no JSONL mutation.
- The secret-prune oracle holds: pruned content is not surfaced by the model on a subsequent turn even when the original message remains physically present in the JSONL.

v0.1.0 ships with verified core behavior. The remaining scenarios (E2E-04 gauge cadence, full E2E-05 missing-JSONL, full Claude Code cold-restart for E2E-06) are flagged for follow-up runs but do not block the release.
